# 零基础转行前端自学指南

# 1.需要重视职业规划的两类人

职业发展的问题和每个人都是非常相关的，但是事实上我接触了上千转行人员和高校毕业生后，发现大多数人几乎没有认真思考过自己的职业规划问题，以至于在工作几年或者毕业找不到工作后，进入IT培训机构，花几万块钱做培训，我正是在他们刚开班的时候了解了这些情况

前面我们说的是大多数人没有思考职业规划问题，那还有一部分人呢？是思考过职业生涯问题的，但由于知识有限、眼界有限以至于选择了一个发展不太好的行业，又或者说在校时候认为非常好的行业，但真正入行以后才发现不是这么回事儿，俗话说，男怕入错行，这话还是有道理的，转行的成本往往是比较大的，也是比较困难的，因为，现在找工作通常都需要工作经验，基于以往的经验去找工作是相对容易的，所以，大多数人一旦进入到某个行业了就有可能会在这个行业里面逗留很久

职业规划对于个人来说是非常重要的，也是非常矛盾的，一方面是因为职业规划会影响你未来几十年的生活，一方面又因为对行业非常的小白，没有清晰的认知，简单的说就是在最需要考虑职业规划问题的时候刚好是自己最无知的时候，很多人就把自己给耽误了

随着互联网的发展，各个行业变化非常快，个人觉得至少两类人非常有必要学习做好个人的职业规划问题：

1.30岁以下需要转行的人

2.高校即将毕业面临找工作的学生

之所以提到上面两类人，原因是这两类人都有一个共同的特点，那就是还有相对较大的空间去重新规划自己的职业生涯，如果是30岁以上了，就会有很多的现实问题，比如结婚、生孩子、车贷、房贷、父母医疗等等，这个时候再考虑换行作为一个新人从头再来的风险太大，会有很多羁绊，另一个方面来说呢，就是公司用新人大多都比较喜欢能干事、能打杂、精力旺盛、可塑性更强的年轻人，从这方面来说换行找个工作成功率相对来说是更低的，这些都是客观上存在的，所以，如果你正青春，就别浪费了，认真的做好自己的职业规划

注意：上面提到的两类人只是说转行相对空间更大，并没有一棍子打死，说30岁以上就一定不能转行

# 2.明确方向：前端开发的发展前景怎样

判断工作的发展前景怎样，我们可以从以下几个维度来考虑：

1.从宏观上来考虑，这个行业总体上有没有更好的发展前景？

2.从这个工作为企业产出的效益来看，有没有可替代性？

3.从工作的工资方面来看，天花板在哪里？

4.从长远发展来看，工作经验是否普遍呈上升趋势？

根据以上几个维度，我们来分析前端开发到底是否适合入坑：

首先，从宏观上来说，随着5G的发展，物联网的发展，以后是万物互联的社会，连接随处可见，有连接的地方就会有界面，有界面的地方就会有前端，前端是与用户最接近的，用户体验对产品来说非常关键，因此，前端的需求也非常多，岗位也会非常多，从这方面将前端的发展还是不错的

其次，从工作为企业产出的效益来说，前端有没有替代性呢？在某些行业是可以被替代的，比如，一个传统制造行业，只需要一个官方网站，找一个建站公司几千块钱就可以搞定了，完全没有必要单独招人开发，如果是在这样的公司，需要居安思危。但是在互联网产品公司，比如，某app公司，一个前端都不需要那是不可能的，至少目前来说不可能

第三，从工资方面来说，前端天花板还是挺高的，优秀的人太少不具有代表性，我们只从转行可以达到的普遍薪资来看，工作几年二线城市15-30万还是属于大概率事件

第四，从长远发展来看，前端编程经验大概率是呈上升趋势的，除非你不思进取技术跟不上，从招聘要求上来看，有多年工作经验的老前端普遍比新人工资高，同时，跳槽一般薪资都会涨一点，如果你从事的行业，跳槽后，和新人没什么差别的工资，那可以毫不犹豫的考虑转行了，比如某电话销售，一般都是按业绩提成，跳槽后几乎是和新人同一起跑线，唯一的优势估计就是业务上更熟悉

# 3.自学编程遇到的困境

如果前面两个问题你都考虑清楚了，决定要转行或者一毕业就要进入前端开发岗位，那么接下来我们就来聊聊如何自学入门前端，并找一个合适的工作。

首先，我们来分析一下自学可能会遇到的问题：

1.知识浩如烟海，课程多如牛毛，对于小白来说，入门的第一问题就是如何从众多课程信息中挑选有用的信息？

2.缺乏完整的学习路径，知识学习的先后顺序不够明确？

3.不够专注，在某宝或者某些论坛可以囤积到几百G或者上T的视频资料，今天学这个，明天学那个如此往复没有一点效果？

4.遇到问题没人答疑，一个问题折腾几天，效率低下，并且打击学习兴趣

5.买了一堆相关书籍，看不懂，或者根本就没有看，有种买了书感觉自己就会了的感觉，实际上浪费了很多时间和金钱

6.计划经常被打乱，学几天间隔两周再回过头来学习，发现前面的忘了，于是又学习一遍，如此往复，效率低下

7.越学越迷茫，学的东西越多感觉自己懂的内容越少

8.看着视频教程，跟着能写，脱离视频教程后感觉自己无法下手

9.学了后面的忘了前面的

10.心态很浮躁，无法沉下心来学习，学习效率低下

# 4.如何筛选高质量的课程

如何挑选高质量课程？可以参考以下几个维度：

1.查看课程大纲，对课程内容做到心中有数，在这一步，你需要关注的就是课程的课时数，和每节课的分钟数，如果每节课程的平均时长超过了20分钟，这种课程一般不建议选择，特别是一些直播课程录制后回放的，这种课程有几个缺点：第一是时长太久，学习一节课程会花费大量时间，很久都学习不完并没有成就感，据非官方研究，人的注意力集中时间为10-15分钟，超过这个时间注意力不集中，学习效率是非常低下的，第二个缺点是时长太长知识点拆分得不够细，回头想要复习的时候，会来回拖动进度条，查找起来也不方便，第三个缺点就是这种高时长课程往往是没有经过后期剪辑处理的，如果是直播课回放的话往往还带有各种无效信息，综上所述，第一个评判维度就是别去选课时非常长的课程

2.了解课程的设计思路，试听讲课的细节，看看这种课程到底是否适合自己。试听课程的时候分两个点来评价，第一，对于基础课程，往往涉及到很多概念性质的东西，那么你就需要搞明白三个问题，这个知识点是什么？这个知识点有什么用？以及这个知识点到底如何去使用？第二，对于实战课程，项目需求是什么，训练的重点是什么？学习后有什么样的收获，这些都需要先搞清楚

3.从课程录制的质量上来说，是否高清？课程是否有杂音？录制的时候是否录制一些无效信息，比如杂乱的桌面、任务栏、无效的操作，例如去打开文件夹的过程等等

4.关注课程本身，而不要一味去追求讲师多牛逼，讲课是一门技术，开发是一门技术，同时具有开发能力和讲课能力的老师在整个IT培训行业都是比较稀缺的，很多属于开发人员出来兼职做讲师的，讲课和课程设计能力有限，还有一些是属于包装得很牛逼的，总之，讲师的title可以做参考，但不是最主要需要考虑的

5.查看课程评价，这个不用多说，相信每个人买东西都会参考

# 6.如何提升学习计划的执行能力？

有些人可能有这样的经历，比如说每次过完年都会制定好当年的计划，到了年底的时候，发现没有几样是完成了的，这里要引出的是计划的科学性和可执行行

制定计划需要注意以下几点：

1、从长远来说，可以确定一个目标，比如：3年后我想买辆车、买套房

2、将目标分解后形成一个短期的计划，这个计划需要有很强的可执行性，也就是说别把目标定得太高，举个例子：我想花一个月减肥100斤，这有点大跃进了

3、在计划完成的时间上，最好设置一个合理的弹性时间

4、计划完成后，给自己一定的奖励，只有自己从心理上认可自己，才能获得成就感，从而更积极的完成自己定的计划

# 7.如何打牢基础，构建前端知识体系？

万丈高楼平地起，勿在浮沙筑高台，所谓的基础知识，就好比是盖房子要打地基一样，没有坚实牢固的地基，房子就没有稳定性，根基不稳，早晚会垮掉。对于零基础的同学来说，打好基础是第一步，有很多机构会以培养编程思维作为课程卖点，但我并不赞同这种方式，我们这边更强调基础知识的重要性，很简单的道理，没有基础就没有思维，编程思维的培养是需要建立在基础知识之上的培养，是掌握了基础后经过大量练习，解决各种问题的过程中，慢慢培养起来的，因此，我们大致把知识分成三类，一类是基础知识，需要记住的，不管你用什么方式，首先得会知道这是什么，得有印象，得记住它，另一类，属于业务性知识，代码写过了，同一个问题可能尝试过各种方案，熟悉各种方案的优缺点，以此选择最优的， 第三类，就是内功修炼，数学、数据结构、算法等等

总结起来就是，基础知识，记住并熟练运用，业务性知识，多写，内功，长期坚持修炼

构建好自己的知识体系，从宏观上做到心中有数，养成这样的好习惯，可以让你在学习后更胸有成竹，也方便你查漏补缺，我们这边一般建议的做法就是自己做思维导图和写博客两种方式

# 8.如何提高解决问题的能力？

提高解决问题的能力并不是一朝一夕的事情，提升编程方面解决问题能力需要做的第一步就是能自己解决bug，有很多初学者自己写的bug自己都搞不定，这是需要提高的，可以这么说，什么时候能自己独立解决自己写的bug，才具有真正的独立自学能力，那么，我们需要从哪几个方面来具体实践，才能慢慢具备这种能力呢？

第一，养成一种习惯，写完代码后运行不理想，首先去看控制台有没有语法错误，这是最低级的错误，一般控制台都会提示你第几行出现错误，你找到报错的地方修改就行，初学者因为语法不熟悉经常会报这种错误

第二，看控制台错误的时候，并不只是看第几行报错，还需要去看具体报错的信息，比如，提示你函数没有定义、对象为空、变量名每定义、少个逗号、分号等等，英语不好的同学最好把这些报错信息用翻译软件翻译过来看看，见过的错误多了，解决起来就更快

第三，如果控制台没有错误，那一般就是逻辑方面写得有问题，比如，改取值的时候没有取到，拿到了一个错误的值进行计算等等，这里就需要你根据业务流程，查看相关的代码执行的顺序来进行调试，在调试的过程中要学会打断点，配合输出语句来确定问题到底出现在哪儿

第四，如果遇到一些未知的，没有见过的错误，比如引入的某个第三方库抛出了错误，或者系统原因的错误，这个时候需要把这些信息拿到搜索引擎里面去搜索，一般情况下，这类错误都具有通用性，比如某个版本升级产生的兼容性错误，是非常方便搜索出来的，也很容易找到解决方案

第五，学会用谷歌浏览器和谷歌搜索，搜索的准确性比较高，效率更高，有人说使用谷歌搜索对提示技术帮助更大，没有人认真证明过这件事，但从我个人使用体验来说，谷歌搜索找答案，的却更高效

# 9.如何做好时间管理？

时间管理的书籍市面上非常的多，我并不是一个很好的时间管理的实践者，也并没有像书中说的那么自律，因此，在这个问题上就不推荐如何去做。多年来，关于时间的习惯我只保持了一个，就是早起，每天4:00钟醒来，晚上12点睡觉，对于我个人而已，早上早起做事更专注，效率更高，所以这个早起这件事情是一定要推荐给大家的，如果4点早起做不到，那你可以做到6点起床，千万别长期睡懒觉，越睡越懒，特别是没有工作的同学，越睡越迷茫
