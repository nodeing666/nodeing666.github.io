# 1. Linux简介

# 1.1.主流操作系统

操作系统是管理计算机软件和硬件资源的系统软件，目前主流的PC操作系统有Windows、Mac、Linux，例如：百度网盘就支持这三个平台

这里移动端的操作系统我们没有讨论，我们找了一份2019年11月份的桌面操作系统市场份额，如下：

从图中我们可以看出，Windows的电脑几乎占比90%以上，Mac系统加起来占比8%左右，所以，咱们日常生活中几乎常见的操作系统就是Windows和Mac，使用桌面Linux系统的人相对来说比较少，因为很多办公软件确实难装

但是，在服务器端操作系统的使用上，Linux的使用量是非常大的，这个可以参考各个云计算厂商的系统选择情况，例如：AWS（亚马逊）、Azure（微软）、阿里云、腾讯云、百度云、金山云、华为云等，这些厂商提供的云服务器操作系统大多数都是Linux版本的，Windows Server从使用量上来说是要大大小于Linux的，造成这种差距的主要的原因还是在于收费，Linux主要是开源免费的（有些付费的版本除外），Windows Server是收费产品

总结一下，大家一定要有一个认知，那就是操作系统不仅仅只有咱们平时笔记本或者台式机上使用的Windows系统和Mac系统，还有在服务器端非常流行的Linux系统，咱们程序员写的程序，大多数都是跑在Linux系统上的，因此，我们才要学习Linux系统，学会如何在Linux系统上部署咱们写好的项目

# 1.2.Linux的版本

linux版本非常的多，也许你听说过Ubuntu、centos、redhat等等，对于初学者来说一听就是懵逼的状态，这里大致的说一下这些版本的分类

linux的版本主要分为两类，一类是linux内核版本，另一类是linux发行版本

linux的内核版本指的是Linux的核心版本，它的核心版本是由Linux内核组织开发维护，发布在这个网站 https://www.kernel.org/

在服务器领域，并不是越新的版本越好，而是越稳定越安全越好

Linux内核版本是向公众免费发放的，谁都可以下载，谁都可以使用，只要用了相同的内核版本，功能就是一样的，如果在这些内核之上再开发一些版本，提供各种不同的功能，这个时候就会变成各个厂商的发行版本，所以简单的说，发行版本就是各个厂商基于内核版本开发的各种版本，有些发行版本是要收费的，例如：redhat的某些功能是需要付费的，目前来说发行版本非常多，你可以在https://distrowatch.com/ 这个网站上查看，参加排名的都有100个发行版本，以下是主流的版本：

在服务器领域redhat版本用得比较多，RHEL（Red Hat Enterprise Linux）的发行方式有两种，1.二进制的发行方式，2.源代码的发行方式。

无论是哪一种发行方式，你都可以免费获得（例如从网上下载），并再次发布。但如果你使用了他们的在线升级（包括补丁）或咨询服务，就必须要付费。

CentOS 就是将 RHEL 发行的源代码从新编译一次，形成一个可使用的二进制版本

由于 LINUX 的源代码是 GNU，所以从获得 RHEL 的源代码到编译成新的二进制，都是合法。只是 REDHAT 是商标，所以必须在新的发行版里将 REDHAT 的商标去掉，因此，CentOS 可以得到 RHEL 的所有功能，甚至是更好的软件。但 CentOS 并不向用户提供商业支持，当然也不负上任何商业责任。

基于以上的原因，我们这门课程使用的Linux系统发行版本是centos，一般服务器从旧不从新，学习用我们可以玩玩centos7，目前生产环境我一般用centos6，如果业务需要升级的时候，评估过后可以升级

# 1.3.环境搭建

# 1.3.1.购买云服务器进行学习

推荐使用腾讯云服务器来做练习机器，目前做活动99/年，【点击购买】，课程中我们使用的练习机器是centos7.5版本

登录腾讯云服务器

Mac系统和windows系统都可以使用ssh的方式登录，都需要在.ssh目录下去生成密钥文件，因此，操作步骤只有一点细微都差别，那就是.ssh目录的存放位置，windows系统和Mac系统的存放位置略有不同，windows下.ssh目录的位置： c:/Users/用户名/.ssh, 以我计算机用户名keymate为例，我的.ssh目录的位置为：c:/Users/keymate/.ssh, 如果你的电脑上没有这个目录，那么你需要自己新建这个目录

Mac系统下.ssh目录：~/.ssh 也就是当前home目录下的.ssh目录

知道以上.ssh目录后，需要生成ssh密钥对

//进入到.ssh目录，执行下面命令
ssh-keygen -t rsa

接下来输入名字，我一般的命名规范是：名字_平台_机器编号，例如：nodeing_tencent_1

名字输入好以后，一直按回车进行下一步操作，直到创建完成,创建完成后，一般会生成两个文件，以nodeing_tencent_1为例，对应的还有一个nodeing_tencent_1.pub文件

接下来，你可以编辑器打开刚才创建的.pub结尾的文件，例如：

然后拷贝这里面的内容到剪贴板，等会儿要粘贴到腾讯云控制台

接下来进入到腾讯云控制台，先把购买的云服务器关机

机器关了以后，找到ssh密钥这个选项

接下来，点创建密钥按钮，弹出的对话框中，选择粘贴已有公钥

上面输入框中，密钥名称可以自定义，输入公钥填写前面几步中.pub文件中的内容

以上步骤做完了，就可以创建好一个密钥，接下来需要把这个密钥和你购买的云服务器绑定

经过上图的两个步骤，密钥和云服务器才绑定成功

接下来，我们需要去配置ssh，这样才可以方便的登录，进入到.ssh这个目录，然后创建一个config的文件，注意不需要带后缀，然后在里面输入如下内容：

# 腾讯云服务器, 这个host 是你登录时候的一个标志
Host nodeing_tencent_1
# HostName是你买服务器的时候，给你分配的一个公网ip
HostName 118.24.40.73
User root
# 这里要填写好生成的密钥文件，注意不带.pub
IdentityFile ~/.ssh/nodeing_tencent_1
ServerAliveInterval 60

配置好以后，可以在终端运行命令：

// nodeing_tencent_1 这个名字就是你在config里面 Host后面定义的名字
ssh nodeing_tencent_1

由于安装虚拟机对电脑配置还是有一定要求，linux系统安装并不是我们这门课程的重点，因此，我们选择了一种比较便捷的方式，直接购买云服务器，这也比较符合目前企业实际的需求，一个项目部署上线，对于大多数企业来说，直接使用云服务器会方便很多，企业服务上云也是大势所趋，安装系统这方面的知识我们就不在这里讲了

# 命令格式

命令名称  [选项] [参数]

举个例子：

ls [选项] [文件或者目录]
选项有：
    -a 显示所有文件，包含隐藏文件
    -l 显示详细信息
    -d 查看目录属性
    -h 人性化显示文件大小
    -i 显示inode
