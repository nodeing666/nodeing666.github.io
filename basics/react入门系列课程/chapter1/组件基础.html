# 1.3.组件基础

学习组件之前，我们先总结前面小节的核心内容：

1. react是什么？react是一个用于构建用户界面的库
2. jsx是什么？就是一种js和html语法混合起来的写法
3. jsx的渲染流程： 通过babel把jsx转换成js语法，通过react创建出一个对象，通过react-dom的render函数把这个对象渲染到html界面中

# 1.3.1.什么是组件？

我们可以把一个复杂的页面分割成各个独立的部分，每个部分都具有独立的逻辑和样式，这些分割出来的独立部分可以通过不同的组合，形成不同的页面，达到代码重用的目的，也方便开发和维护，这些分割出来的独立部分就是组件，例如：

# 1.3.2.组件的定义方式

1.方式1: 通过函数定义组件

import React from 'react'
import ReactDOM from 'react-dom'

let Hello = ({msg}) => {
  return <h1>{msg}</h1>
}

ReactDOM.render(
  <Hello msg="hello nodeing!!!" />,
  document.getElementById('root')
)

上面代码中，Hello组件的msg属性 会被传给组件定义是的参数，这个参数是一个对象，可以修改一下代码来验证：

let Hello = (props) => {
  console.log(props)
  return <h1>props.msg</h1>
}

注意：组件的首字母一定要写成大写字母

2.方式2:通过类的方式来定义组件

第一种方式定义的组件非常简单，属于静态的组件，没有交互的代码，如果一个组件交互代码非常多，那么第一种方式就不能满足我们的需求，因此，我们需要通过类的方式来定义组件，实际上在实际react开发中，常用的还是这种方法来定义组件

import React from 'react'
import ReactDOM from 'react-dom'

// 第一种方式
// let Hello = (props) => {
//   return <h1>{props.msg}</h1>
// }
//  第二种方式
class Hello extends React.Component {
  constructor () {
    super()
    // 定义组件的状态
    this.state = {num: 0}
  }
  // 组件挂载前执行这个函数
  componentDidMount () {
    window.setInterval( () => {
      // 设置组件的状态
      this.setState({num: Math.random()* 1000})
    }, 1000)
  }
  // 类中必须有一个render函数，表示如何去渲染组件，并且必须返回一个react元素（写法上就是一个html标签）
  render () {
    return <h1>{this.state.num}</h1>
  }
}

ReactDOM.render(
  <Hello msg="hello nodeing!!!" />,
  document.getElementById('root')
)

# 1.3.3.组件的状态

关于组件的状态，这里需要掌握两点；1 如何去定义状态 2 如何去修改状态，实际上react中组件的状态就是用来存数据的

定义状态：

constructor () {
    super()
    // 定义组件的状态
    this.state = {num: 0}
}

修改状态：

window.setInterval( () => {
  // 设置组件的状态
  this.setState({num: Math.random()* 1000})
}, 1000)

# 1.3.4.组件的属性

1.属性本质上是用来描述某个事物的特征的，react组件的属性和html属性的用法一样，都是在调用的时候定义在标签上的，例如：

<h1 id="class">hello</h1>
<Hello name="nodeing" />

2.如何去获取属性呢？在组件内部，可以通过this.props获取到属性

import React from 'react'
import ReactDOM from 'react-dom'

class Hello extends React.Component {
  render () {
    return <h1>{this.props.msg}</h1>
  }
}

ReactDOM.render(
  <Hello msg="hello nodeing!!!" />,
  document.getElementById('root')
)

3.什么是默认属性？当组件调用的时候没有传人属性，那么就可以使用默认属性，如果传入了属性，就使用传入的这个属性，定义默认属性的方式如下：

import React from 'react'
import ReactDOM from 'react-dom'

class Hello extends React.Component {
  static defaultProps = {
    msg: '螺钉课堂！！！'
  }
  render () {
    // 使用默认msg属性
    return <h1>{this.props.msg}</h1>
  }
}

ReactDOM.render(
  // 这里没有传入msg
  <Hello  />,
  document.getElementById('root')
)

4.什么是属性类型检查？当你的应用日渐庞大，类型检查可以让你避免更多的错误，说白了就是规定你传入的属性必须是什么类型，是否必须传入等

举个例子：

首先需要安装一个类型检查库，prop-types

npm i prop-types --save

import React from 'react'
import ReactDOM from 'react-dom'
import PropTypes from 'prop-types'
class Hello extends React.Component {
  static defaultProps = {
    msg: '螺钉课堂！！！'
  }
  static propTypes = {
    msg: PropTypes.string
  }
  render () {
    // 使用默认msg属性
    return <h1>{this.props.msg}</h1>
  }
}

ReactDOM.render(
  // 这里没有传入msg
  <Hello msg={100} />,
  document.getElementById('root')
)

上面代码中，类型检查器规定msg是一个字符串类型，但是传入的时候是一个数字，这个时候会有警告提醒

上面只是演示了如何去规定一个类型，按照这种操作方式，你可以规定各种类型，具体用法可以看下面的代码，用到的时候查文档就行，不需要记住这么多代码

MyComponent.propTypes = {
  // 你可以将属性声明为以下 JS 原生类型
  optionalArray: PropTypes.array,
  optionalBool: PropTypes.bool,
  optionalFunc: PropTypes.func,
  optionalNumber: PropTypes.number,
  optionalObject: PropTypes.object,
  optionalString: PropTypes.string,
  optionalSymbol: PropTypes.symbol,

  // 任何可被渲染的元素（包括数字、字符串、子元素或数组）。
  optionalNode: PropTypes.node,

  // 一个 React 元素
  optionalElement: PropTypes.element,

  // 你也可以声明属性为某个类的实例，这里使用 JS 的
  // instanceof 操作符实现。
  optionalMessage: PropTypes.instanceOf(Message),

  // 你也可以限制你的属性值是某个特定值之一
  optionalEnum: PropTypes.oneOf(['News', 'Photos']),

  // 限制它为列举类型之一的对象
  optionalUnion: PropTypes.oneOfType([
    PropTypes.string,
    PropTypes.number,
    PropTypes.instanceOf(Message)
  ]),

  // 一个指定元素类型的数组
  optionalArrayOf: PropTypes.arrayOf(PropTypes.number),

  // 一个指定类型的对象
  optionalObjectOf: PropTypes.objectOf(PropTypes.number),

  // 一个指定属性及其类型的对象
  optionalObjectWithShape: PropTypes.shape({
    color: PropTypes.string,
    fontSize: PropTypes.number
  }),

  // 你也可以在任何 PropTypes 属性后面加上 `isRequired` 
  // 后缀，这样如果这个属性父组件没有提供时，会打印警告信息
  requiredFunc: PropTypes.func.isRequired,

  // 任意类型的数据
  requiredAny: PropTypes.any.isRequired,

  // 你也可以指定一个自定义验证器。它应该在验证失败时返回
  // 一个 Error 对象而不是 `console.warn` 或抛出异常。
  // 不过在 `oneOfType` 中它不起作用。
  customProp: function(props, propName, componentName) {
    if (!/matchme/.test(props[propName])) {
      return new Error(
        'Invalid prop `' + propName + '` supplied to' +
        ' `' + componentName + '`. Validation failed.'
      );
    }
  },

  // 不过你可以提供一个自定义的 `arrayOf` 或 `objectOf` 
  // 验证器，它应该在验证失败时返回一个 Error 对象。 它被用
  // 于验证数组或对象的每个值。验证器前两个参数的第一个是数组
  // 或对象本身，第二个是它们对应的键。
  customArrayProp: PropTypes.arrayOf(function(propValue, key, componentName, location, propFullName) {
    if (!/matchme/.test(propValue[key])) {
      return new Error(
        'Invalid prop `' + propFullName + '` supplied to' +
        ' `' + componentName + '`. Validation failed.'
      );
    }
  })
};

# 1.3.5.给组件绑定事件处理函数

import React from 'react'
import ReactDOM from 'react-dom'
class Hello extends React.Component {
  constructor () {
    super()
    this.state = {
      num: 0
    }
    
  }
  handleClick = () => {
    let num2 = this.state.num
    num2++
    this.setState({num: num2})
  }
  render () {
    // 使用默认msg属性
    return <h1 onClick={this.handleClick}>{this.state.num}</h1>
  }
}

ReactDOM.render(
  // 这里没有传入msg
  <Hello  />,
  document.getElementById('root')
)
