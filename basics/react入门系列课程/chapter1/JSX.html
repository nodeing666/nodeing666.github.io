# 1.2.JSX语法

# 1.2.1.什么是JSX语法

jsx官方定义是类XML语法的ECMAScript扩展，它使用JS语法和HTML语法来创建虚拟元素，也就是说JSX语法，从写法上来说，就是你可以在js代码中，直接以html语法来创建元素

ReactDOM.render(
  <h1>hello nodeing!!!</h1>,
  document.getElementById('root')
)

# 1.2.2.JSX基本原理

为什么可以直接在js文件里面混合html语法呢？React内部会构建一个Javascript对象来描述你的html信息，这个对象里面包括标签名称、属性、子元素等，这样就符合js语法了，然后再通过render函数塞到网页中，本质上，都是Javascript对象

我们可以定义一个虚拟元素，然后在浏览器里面打印出来看看

import React from 'react'
import ReactDOM from 'react-dom'

// 定义一个虚拟元素
let element = <h1>hello nodeing!!!</h1>

打印结果

上面这种定义虚拟元素到方式，是通过React.createElement实现的

// 定义虚拟元素
let element2 = React.createElement('h1', {id: 'box'}, ['hello nodeing!!!'])
console.log(element2)

createElement方法就是用来创建虚拟元素的，有三个参数，第一个参数是标签类型，第二个参数是属性对象，第三个参数是子元素

打印结果

我们可以直接把element2渲染到页面中

import React from 'react'
import ReactDOM from 'react-dom'

// 定义一个虚拟元素
// let element = <h1>hello nodeing!!!</h1>
// console.log(element)

// 定义虚拟元素
let element2 = React.createElement('h1', {id: 'box'}, ['hello nodeing!!!'])
console.log(element2)

ReactDOM.render(
  element2,
  document.getElementById('root')
)

从上面的代码，我们知道了，不管是以什么样的方式创建的虚拟元素，最终得到的都是一个JavaScript对象，这一步操作是由react.js和Babel帮忙完成的, 可以尝试将代码放到Babel中运行看看，点击运行

接下来就是react-dom来做渲染了，react-dom里面有一个render方法，会把前面说到的JavaScript对象渲染到DOM中，为了说明原理，我们简单的手写一个render函数来解释

假设需要渲染一个h2元素到root容器中

<h2>hello nodeing!!!</h2>

这个h2元素最终可以用下面的对象来描述

let oH2 = {
  type: 'h2',
  props: {
    id: 'title',
    children: ['hello nodeing!!!']
  }
}

接下来，我们需要去定义一个render函数来把上面的对象渲染到root容器中

function render(eleObj, container) {
  let {type, props} = eleObj
  let ele = document.createElement(type)
  for( let attr in props) {
    if (attr !== 'children') {
      ele.setAttribute(attr, props[attr])
    } else {
      // 假设只有一个文本节点
      ele.innerText = props.children
    }
  }
  container.appendChild(ele)
}

完成以上步骤，咱们的虚拟元素对象就被渲染到html中了，注意；为了简单说明原理，render方法并不完善
