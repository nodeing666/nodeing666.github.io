# 2.js面向对象进阶和组件开发

# 2.1.包装对象

前面我们学习过字符串的相关方法，例如：charAt、indexOf、replace等，那大家可以想想，字符串到底是什么类型？为什么字符串会有方法？

var str1 = "nodeing.com"
//string类型
alert(typeof str1)
//为什么string类型会有charAt方法呢？str1并不是一个对象？
console.log(str1.charAt(0))

要解决上面的问题，就要理解包装对象的相关知识，基本类型都有其对应的一个包装对象，例如：字符串类型对应的包装对象是String，数字的包装对象是Number，布尔类型对应的包装对象是Boolean

通过new String来创建字符串

var str1 = new String("nodeing")
alert(typeof str1)  //通过new String创建出来的str是一个对象
console.log(str1.charAt(0))

基于前面我们学过的原型知识，我们可以猜测一下charAt可能是挂载到String构造函数原型上的，所以当str对象去调用charAt方法是可以正常运行的

String.prototype.charAt = function(){}

接下来，我们给String构造函数自定义一个方法，看普通的字符串是否也能够使用

//创建一个字符串对象
var str1 = new String("nodeing")
String.prototype.test = function(){
    alert(1)
}

//创建一个基本类型字符串
var str2 = "nodeing"
str2.test()  // 弹出1

//如果给str2挂载一个属性，看看效果
str2.num = 10
alert(str2.num) // undefined

从上面的代码我们知道了 基本类型的str2可以使用方法，实际上使用的是String身上的方法

var s1 = "nodeing"
s1.charAt(0)

上面代码实际上执行了以下几个步骤：

1.创建String 类型的对象
2.在对象上调用指定的方法
3.销毁这个对象

总结： 1、字符串、数字、布尔值分别对应了String、Number、Boolean等包装对象 2、基本类型调用的方法实际上是包装对象上的 3、基本类型方法调用后包装对象会消失

# 2.2.原型链

原型链就是实例对象和原型之间的链接，回顾一下我们学过的原型知识，每个函数都有一个prototype属性，这个prototype指向一个对象，这个对象叫做原型，通过构造函数创建出来的实例也有一个内部指针指向原型，这样就形成实例对象和原型直接的链接，我们把这个链接称为原型链

function Fn1(){
    
}
Fn1.prototype.num = 20
var f1 = new Fn1()
alert(f1.num)  //20

图上f1和Fn1.prototype之间建立的链接就是原型链，在f1内部的指针用[[prototype]]来表示，原型链是不可见的，但是，大部分浏览器中已经实现了，给这条原型链取了个英文名字叫“__proto__”,通过这个属性可以查看和设置原型对象,在es5中这并不是标准，在es6中想查看原型对象可以用getPrototypeOf()方法，想设置原型对象用setPrototypeOf方法


function Fn1(){}
Fn1.prototype.num = 20
var f1 = new Fn1()
console.log(Fn1.prototype)
console.log(f1.__proto__)
console.log(Object.getPrototypeOf(f1))

原型链有多层，最外层为Object.prototype

function Fn1(){}
Fn1.prototype.num = 20
Object.prototype.num = 30
var f1 = new Fn1()
console.log(f1.__proto__)
console.log(f1.__proto__.__proto__.num)  //30

注意：访问属性的时候，是根据就近原则访问的，例如：f1，Fn.prototype, Object.prototype上都有num属性，那么alert(f1.num)的时候，首先取出的是f1身上的值，如果f1身上没有，那么才访问Fn.prototype,以此类推，访问到最外层Object.prototype，如果最终都没有就返回undefined

# 2.3.面向对象的相关属性和方法

# 2.3.1.hasOwnProperty

看看是不是对象自身的属性

function Fn1(){
    this.num = 10
}
Fn1.prototype.num2 = 20

var f1 = new Fn1()

alert(f1.hasOwnProperty("num"))  //true

# 2.3.2.constructor

无论什么时候，只要创建了一个新的函数，就会根据一组特定的规则为该函数创建一个prototype属性，这个属性指向函数的原型对象，在默认情况下，所有原型对象都会自动获得一个constructor属性，这个属性指向prototype属性所在函数的指针，总结起来说就是 在原型对象上默认有个constructor属性是指向构造函数的

function Fn1(){}
Fn1.prototype.num = 10
console.log(Fn1.prototype.constructor)

因为原型对象上的属性和方法，每个被构造函数创建出来的对象都能访问，因此，我们在具体的对象上就可以通过constuctor来查看构造函数，意味着我们能知道某个对象是由哪个构造函数创建出来的

function Fn1(){}
var f1 = new Fn1()
console.log(f1.constructor) //Fn1

能查看构造函数，有时候可以用来做一些简单的判断

//判断一个变量是不是数组
var arr = []
console.log(arr.constructor === Array) // true

注意：避免修改constructor,constructor是挂载到原型对象上的，意味着当一个构造函数新创建出来后，会自动生成下面一句话

function Fn1(){}
Fn1.prototype.constructor = Fn1

试着修改一下constructor

function Fn1(){}
Fn1.prototype.constructor = Array
var f1 = new Fn1()
console.log(f1.constructor)  //Array

如果constructor被修改了，会产生各种未知的错误，因此，最好避免不要修改它，如果有些时候不经意间修改了，可以把这个指向修正一下

function Fn1(){}
Fn1.prototype.name = 10
Fn1.prototype.age = 20

如果觉得上面写法比较麻烦，可能会改成这样：

function Fn1(){}
Fn1.prototype = {
    name: 10,
    age: 20
}
// 指向Object
console.log(Fn1.prototype.constructor)

上面的写法不经意间就会把constructor修改掉，因此，我们需要手动去修正

function Fn1(){}
Fn1.prototype = {
    constructor: Fn1,
    name: 10,
    age: 20
}
// 指向Object
console.log(Fn1.prototype.constructor)

注意：有些属性不能用for in语句循环出来

function Fn1(){}
Fn1.prototype.num = 10
for(var attr in Fn1.prototype){
    console.log(attr)   //num
}

自己添加的num属性是可以被打印出来的，系统自带的没办法for in循环出来

# 2.3.3.instanceof

instanceof是一个运算符，用来测试一个对象在其原型链中是否存在一个构造函数的 prototype 属性。检测一个对象和构造函数是否在同一个原型链上

function Fn1(){}
var f1 = new Fn1()
alert(f1 instanceof Fn1)  //true
alert(f1 instanceof Array) //false

利用这个运算符也可以做简单的类型判断

var arr = []
alert(arr instanceof Array) //true

# 2.3.4.toString

toString的作用是把对象转换成字符串，如果是系统自带的对象去调用toString方法，那么toString方法在原型对象上，如果不是系统自带的对象去调用toString方法，那么toString在Object.prototype上

var arr = [2,2,3]
console.log(typeof arr.toString()) //string
console.log(arr)  // 在浏览器中__proto__可以查看到toString方法
//验证toString位置
alert(arr.toString === Array.prototype.toString)

function Fn1() {}
var f1 = new Fn1()
console.log(f1)  //toString 在Object.prototype上

//验证toString位置
alert(f1.toString === Object.prototype.toString)

toString也可以用来把数字转成16进制字符串

var arr = 255
console.log(arr.toString(16))

toString还可以做类型判断

var arr = []
console.log(Object.prototype.toString.call(arr)); // object Array
alert(Object.prototype.toString.call(arr) === '[object Array]');

前面我们已经学过两种判断类型的方法，这里推荐第三种toString的形式，这也是很多大的库里面使用的方法，为什么推荐使用toString来判断类型？具体可以参考这片文章https://github.com/lifesinger/blog/issues/175,也可以参考下面的代码

# 2.4.继承

# 2.4.1.什么是继承

在编程中，继承就是子类去继承父类的属性和方法，类的概念中，我们说类就是归类的意思，在一大类中可以分出很多小类，例如，动物类，属于大类，动物类下面可以分为狗类和人类，其中，人类就是动物类的子类，狗类下面又可以分很多类，以此类推

类的关系

这里需要强调的是父与子的关系，这也类似于生活中的父子关系，生活中的继承儿子可以继承父亲的财产，编程中的继承子类可以继承父类的属性和方法，具体表现就是在原有对象的基础上稍作修改就可以得到一个新的对象，因此，继承的作用就是复用代码，节约代码,说白了继承就是一种复用代码的形式

# 2.4.2.call、apply方法

call方法和apply方法类似，我们着重讲解call方法，call方法表示调用一个对象的一个方法，以另一个对象替换当前对象，可以理解为借用一个对象的方法

function Person1(name) {
				this.name = name
            }
Person1.prototype.showName = function () {
    alert(this.name)
}
var p1 = new Person1("小红")

function  Person2(){}
var p2 = new Person2()

//p2上并没有showName方法 但是可以去借p1的来用
//翻译一下：p1有一个showName方法，借给了p2
p1.showName.call(p2)  //undefined   因为p2里面什么都没有 this.name就为undefined

call方法是可以带参数的

function Person1(name) {
				this.name = name
            }
Person1.prototype.showName = function (name, age) {
    alert(name)
    alert(age)
}
var p1 = new Person1("小红")

function  Person2(){}
var p2 = new Person2()

//p2上并没有showName方法 但是可以去借p1的来用
//翻译一下：p1有一个showName方法，借给了p2,p2在用的时候传入两个参数
p1.showName.call(p2, "小张", 200)

call方法和apply方法功能是非常相似的，区别就是传参方式不一样

p1.showName.call(p2, "小张", 200)
p1.showName.apply(p2, ["小张", 200])

# 2.4.3.继承初识

在没有使用继承方式写代码之前，我们的代码可能是这样的

function Parent(name, age) {
    this.name = name
    this.age = age
}
Parent.prototype.showName = function () {
    alert(this.name)
}
function Child(name, age) {
    this.name = name
    this.age = age
}
Child.prototype.showName = function () {
    alert(this.name)
}
Child.prototype.showAge = function () {
    alert(this.age)
}

从上面的代码中我们可以看出，有些代码是重复了的，在写Child构造函数的时候，name属性和age属性是可以直接使用Parent里面的，既然Parent是一个函数，那么我们可以直接在Child里面调用就可以了

function Child(name, age) {
    Parent(name,age)
}

但是直接这样写还不行，Parent里面的this指向会出现问题，直接调用Parent()函数，相当于window.Parent(),那么函数里面的this是指向window的，这个时候，我们需要修正Parent里面的this，让Parent里面的this指向Child才行，因此，我们的代码需要写成这样:

function Child(name, age){
    Parent(this, name, age)
}

通过这种函数调用的形式，我们可以直接把Parent里面的属性都继承过来，但是，Parent上面的方法该如何继承呢？最简单的方式就是把Parent的原型赋值给Child原型

function Parent(name, age) {
    this.name = name;
    this.age = age;
}
Parent.prototype.showName = function () {
    alert(this.name)
};
function Child(name, age) {
    Parent.call(this, name, age)
}
Child.prototype = Parent.prototype;
Child.prototype.showAge = function () {
    alert(this.age)
};
var c1 = new Child("小红", 222)
c1.showName()
c1.showAge()

# 2.4.4.继承方式改进

前面我们谈到继承方法的时候，直接把Parent的原型赋值给了Child的原型，这种方法简单粗暴，但是会有一些影响？继承的一个特点是，儿子去继承父亲的属性和方法，但是不要对父亲有任何的影响，我们如果直接把Parent的原型赋值给Child的原型，这样实际上是对象的引用，这样的话，他们就会相互影响

Child.prototype = Parent.prototype;

像这样对象引用会造成相互影响，那我们解决这个问题的关键就是不让它们产生引用关系，由此，我们联想到了基本类型的赋值，例如：

Child.prototype.num = Parent.prototype.num
Child.prototype.age = Parent.prototype.age
Child.prototype.showName = Parent.prototype.showName
Child.prototype.showAge = Parent.prototype.showAge
......

上面代码中，我们只是把Parent.prototype这个对象里面的属性，例如，num、age等，直接赋值给了Child.prototype这个对象上的属性，这样一项一项的赋值就是基本类型的赋值，不会产生引用关系，这里要强调的是，把函数赋值给另一个对象的属性是不会相互影响的，因为，函数块本身是不能被修改的，例如，下面这句：

# 这里的showAge虽然是函数，但是赋值后因为函数不能被修改，所以，可以直接赋值
Child.prototype.showAge = Parent.prototype.showAge

但是，我们并不清楚也不确定Parent.prototype会有多少属性和方法，直接一项一项的赋值会非常的麻烦，因此，我们可以写一个简单的方法来实现

function extend(Child, Parent){
    for(var attr in Parent){
        Child[attr] = Parent[attr]
    }
}

最终，总结一下继承的方式：1 继承属性，通过call的方式实现 2 继承方法，通过for循环拷贝的方式实现

function Parent(name, age) {
    this.name = name;
    this.age = age;
}
Parent.prototype.showName = function () {
    alert(this.name)
};
function Child(name, age) {
    Parent.call(this, name, age)
}
// Child.prototype = Parent.prototype; 把这句换掉
extend(Child.prototype, Parent.prototype)

Child.prototype.showAge = function () {
    alert(this.age)
};

function extend(Child, Parent){
    for(var attr in Parent){
        Child[attr] = Parent[attr]
    }
}

# 2.4.5.练习

需求：
    1.父类是一个可以拖拽的div盒子，可以在任意范围内拖拽
    2.子类去继承父类，要求具有父类的全部功能，自己特有的功能是不能拖出可视区

实现代码:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
    <style>

        }

<div id="box2"></div>
<script>
    function Drag(id) {
        this.oBox = document.getElementById(id);
        this.difX = 0;
        this.difY = 0;

    }
    Drag.prototype.init = function () {
        var That = this;
        this.oBox.onmousedown = function (ev) {
            That.fnDown(ev)
        }
    };
    Drag.prototype.fnMove = function (ev) {
        var ev = ev || window.event;
        this.oBox.style.left = ev.clientX - this.difX + "px";
        this.oBox.style.top = ev.clientY - this.difY + "px";

    };
    Drag.prototype.fnUp = function () {
        document.onmouseup = null;
        document.onmousemove = null;
    };
    Drag.prototype.fnDown = function () {
        var ev  = ev || window.event;
        var That = this;
        //获取到差值
        this.difX = ev.clientX - this.oBox.offsetLeft;
        this.difY = ev.clientY - this.oBox.offsetTop;

        document.onmousemove = function (ev2) {
            //移动的功能
            That.fnMove(ev2)
        }
        document.onmouseup = function (ev2) {
            That.fnUp(ev)
        }
    };
    //通过继承实现代码复用
    function ChildDrag(id) {
        Drag.call(this, id)
    }
    extend(ChildDrag.prototype, Drag.prototype);

    ChildDrag.prototype.fnMove = function (ev) {
        var ev = ev || window.event;
        var L = ev.clientX - this.difX;
        var T = ev.clientY - this.difY;
        if(L < 0){
            L = 0;
        }
        if(L > document.documentElement.clientWidth - this.oBox.offsetWidth){
            L = document.documentElement.clientWidth - this.oBox.offsetWidth
        }
        if(T < 0){
            T = 0;
        }
        if(T > document.documentElement.clientHeight - this.oBox.offsetHeight){
            T = document.documentElement.clientHeight - this.oBox.offsetHeight
        }
        console.log(L)
        this.oBox.style.left = L + "px";
        this.oBox.style.top =  T + "px";
    }
    function extend(child, parent) {
        for(var attr in parent){
            child[attr] = parent[attr]
        }
    }

    var d1 = new Drag("box1");
    d1.init();

    var d2 = new ChildDrag("box2");

# 2.5.组件开发

# 2.5.1.组件开发是什么

组件开发其实也是一种代码复用的形式，它和继承的方式复用代码还是有些区别，继承主要表现为子类去继承父类，子类的实现可能有些地方和父类不一样，但是一样的东西可以拿过来用，达到复用代码的效果，而组件开发的方式复用代码，更像是兄弟之间的关系，是一种更丰富的代码复用形式，例如：

弹窗1:

弹窗2:

还有各种各样的弹窗，这些形式非常丰富的功能，每种形式都去写一套代码就会显得臃肿，因此，我们需要用组件的形式来复用代码，用同一套代码来满足不同的产品需求

总结起来，组件开发本质上就是一种代码复用形式，基于一套代码来满足不同的需求
