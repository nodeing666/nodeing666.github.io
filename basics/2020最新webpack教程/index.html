# webapack最佳入门教程

# 1.webpack相关概念

# 1.1.什么是webpack？

webpack 是一个现代 JavaScript 应用程序的静态模块打包器(module bundler)。

# 1.2.打包初体验

第一步：安装webpack，由于webpack是建立在Nodejs上的，因此，在具体安装webpack之前，你需要先安装nodejs，如果你还不知道nodejs是什么，那么先不要往下学习，先去把nodejs基础学了再来，以免浪费时间

我们这里假设你已经安装了nodejs

//1. 创建一个webpack-demo的项目文件夹
mkdir webpack-demo 

// 2. 初始化项目
npm init -y

// 3. 在本地安装 webpack 这里我们指定版本4.42.1
npm i webpack@4.42.1 

// 4. 安装webpack-cli 这里我们指定
npm i webpack-cli@3.3.11

第二步：在项目下新建一个src目录，然后在src目录下新建index.js文件,输入下面的代码

let a = 20
alert(a)

第三步：修改package.json文件

{
  "name": "webpack-demo",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    // 注意这里的代码是增加的，不要直接copy我这里的代码，package.json里面不能添加注释
    "dev": "webpack"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "dependencies": {
    "webpack": "^4.42.1",
    "webpack-cli": "^3.3.11"
  }
}

第四步：运行下面命令

npm run dev

执行上面的命令后，你会发现 webpack-demo这个目录下，新增了一个dist目录，并且生成了一个 main.js的文件，这个文件默认被压缩了，这就是webpack这个工具的作用

执行完上面的步骤，相信你已经对webpack有一定的认知了，我们再在这里总结一下：webpack是什么？webpack就是一个模块打包工具，它可以把js、图片、html等经过相应的处理(比如压缩，合并等)，最终生成线上可部署的静态资源

# webpack作为构建工具，可以做很多事情

1.代码转换： TypeScript编译成JavaScript、SCSS编译成CSS等
2.文件优化： 压缩JavaScript、CSS、HTML代码，压缩合并图片等
3.代码分割：提取多个页面的公共代码、提取首屏不需要执行部分的代码让其异步价值
4.模块合并：在采用模块化的项目里会有很多模块和文件，需要构建功能把模块分类合并成一个文件
5.自动刷新：监听本地源代码的变化，自动重新构建、刷新浏览器
6.自动校验：在代码被提交到仓库前需要校验代码是否符合规范、以及单元测试是否通过
7.自动发布：更新完代码后，自动构建出线上发布代码并传输给发布系统

# 1.3.核心概念

前面我们初体验打包，必须按照webpack的规则来新建src目录，js必须写成index.js 这样就显得不是太灵活，如果我们想要更灵活的使用webpack，我们需要对其进行配置，并且了解相关的概念，下面的概念，可以先了解，我们会在配置文件里面详细讲到

# 入口(Entry)

入口起点(entry point)指示 webpack 应该使用哪个模块，来作为构建其内部依赖图的开始。进入入口起点后，webpack 会找出有哪些模块和库是入口起点（直接和间接）依赖的。

# 出口(output)

output 属性告诉 webpack 在哪里输出它所创建的 bundles，以及如何命名这些文件，默认值为 ./dist。基本上，整个应用程序结构，都会被编译到你指定的输出路径的文件夹中

# loader

loader 让 webpack 能够去处理那些非 JavaScript 文件（webpack 自身只理解 JavaScript）。loader 可以将所有类型的文件转换为 webpack 能够处理的有效模块，然后你就可以利用 webpack 的打包能力，对它们进行处理。通俗的说，loader就是转换器，用于把模块原内容按需求转行成新内容，默认情况下webpack只识别js模块，如果是非js模块就需要使用loader来转换

# 插件(plugins)

loader 被用于转换某些类型的模块，而插件则可以用于执行范围更广的任务。插件的范围包括，从打包优化和压缩，一直到重新定义环境中的变量。插件接口功能极其强大，可以用来处理各种各样的任务。插件就是在webpack构建流程中的特定时机注入扩展逻辑来改变构建结果，可以通俗的理解为扩展功能

# 模式(mode)

通过选择 development 或 production 之中的一个，来设置 mode 参数，你可以启用相应模式下的 webpack 内置的优化

# 2.自定义配置

了解了前面的一些概念后，我们可以自定义webpack的配置了，我们需要在webpack-demo这个项目的根目录下面，去创建一个webpack.config.js的配置文件

// 引入nodejs的path模块来处理路径
const path = require("path")
module.exports = {
  // 模式配置
  mode: "development",
  // 入口文件  通常使用绝对路径
  entry: path.resolve(__dirname, "./src/index.js"),
  // 出口配置
  output: {
    filename: "app.js",
    path: path.resolve(__dirname, "dist")
  }
}

# 3.webpack-dev-server

前面我们通过配置，可以使用命令将修改的代码打包出来，如果这是在开发阶段会比较麻烦，我们希望能更方便的修改代码，并且在浏览器中实时刷新，webpack-dev-server就是帮我们做这样一件事情的，webpack-dev-server是我们在开发阶段需要用到的一个服务器,它会把代码打包到内存，我们可以通过http的方式访问到打包到内存的代码，当源代码修改了的时候，会自动重启服务器并且刷新代码显示最新的结果

第一步：安装webpack-dev-server

npm i webpack-dev-server@3.10.3

第二步：修改webpack.config.js 配置文件

// 引入nodejs的path模块来处理路径
const path = require('path')
module.exports = {
  // 模式配置
  mode: 'development',
  // 入口文件  通常使用绝对路径
  entry: path.resolve(__dirname, './src/index.js'),
  // 出口配置
  output: {
    filename: 'app.js',
    path: path.resolve(__dirname, 'dist')
  },
  devServer: {
    // 服务器的根路径对应的目录
    contentBase: path.resolve(__dirname, 'dist')
    // 这里指定端口号
    port: 8080,
    // 服务器启动后自动打开浏览器
    open: true
  }
}

第三步： 修改package.json文件

//
{
  "name": "webpack-demo",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    // 这里把原来的webpack 修改成 webpack-dev-server
    "dev": "webpack-dev-server"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "dependencies": {
    "webpack": "^4.42.1",
    "webpack-cli": "^3.3.11",
    "webpack-dev-server": "^3.10.3"
  }
}

# 4.自动创建html文件

在前面webpack-dev-server中，我们设置的启动访问路径是项目下的dist目录，我们需要在这个dist目录下手动去创建一个index.html文件，并且将打包后的js文件手动添加到index.html文件中，这个时候，我们通过浏览器访问才能看到效果,在开发中，能用工具做的事情我们尽量用工具帮我们完成，我们可以使用 html-webpack-plugin 这个插件来帮我们做这件事情，它的作用就是根据模版文件生成一个静态文件，然后自动将打包好的js插入到这个 html文件中

首先，安装这个插件

npm i html-webpack-plugin@4.0.2

修改webpack.config.js文件

// 引入nodejs的path模块来处理路径
const path = require('path')
const htmlWebpackPlugin = require('html-webpack-plugin')
module.exports = {
  // 模式配置
  mode: 'development',
  // 入口文件  通常使用绝对路径
  entry: path.resolve(__dirname, './src/index.js'),
  // 出口配置
  output: {
    filename: 'app.js',
    path: path.resolve(__dirname, 'dist')
  },
  devServer: {
    contentBase: path.resolve(__dirname, 'dist'),
    open: true
  },
  plugins: [
    new htmlWebpackPlugin({
      // 设置标题
      title: '螺钉课堂',
      // 设置模版路径  注意，这里你需要在你的项目目录下 新建一个public文件夹来存放 html模版
      template: path.resolve(__dirname, 'public/index.html'),
      // 设置打包的文件名
      filename: 'index.html',
      // 最小输出配置
      minify: {
        // 折叠空格
        collapseWhitespace: true
      }
    })
  ]
}

运行查看效果

npm run dev

# 5.自动清除上次打包的文件

当我们执行打包操作，自动生成一个打包文件，如果前面有已经生成的文件，我们通常会手动去删除前一次已经生成的文件，以避免产生混乱，这个时候可以使用clean-webpack-plugin来帮我们完成这个操作

安装插件

npm i clean-webpack-plugin@3.0.0

修改配置文件


// 引入nodejs的path模块来处理路径
const path = require('path')
const htmlWebpackPlugin = require('html-webpack-plugin')
const { CleanWebpackPlugin } = require('clean-webpack-plugin')
module.exports = {
  // 模式配置
  mode: 'development',
  // 入口文件  通常使用绝对路径
  entry: path.resolve(__dirname, './src/index.js'),
  // 出口配置
  output: {
    filename: 'app.js',
    path: path.resolve(__dirname, 'dist')
  },
  devServer: {
    contentBase: path.resolve(__dirname, 'dist'),
    open: true
  },
  plugins: [
    new htmlWebpackPlugin({
      // 设置标题
      title: '螺钉课堂',
      // 设置模版路径
      template: path.resolve(__dirname, 'public/index.html'),
      // 设置打包的文件名
      filename: 'index.html',
      // 最小输出配置
      minify: {
        // 折叠空格
        collapseWhitespace: true
      }
    }),
    // 每次打包前 先清空上次打包的目录
    new CleanWebpackPlugin()
  ]
}

修改package.json 去配置一个build命令

{
  "name": "webpack-demo",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "dev": "webpack-dev-server",
    // 这里是新增加的命令
    "build": "webpack"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "dependencies": {
    "clean-webpack-plugin": "^3.0.0",

# 6.解析css文件

当我们在js文件中引入css文件的时候会报错，例如：

index.js 文件内容

let a = 20
alert(a)
// 引入css文件
import './index.css'

这个时候执行npm run dev会把报错

ERROR in ./src/index.css 1:5
Module parse failed: Unexpected token (1:5)
You may need an appropriate loader to handle this file type, currently no loaders are configured to process this file. See https://webpack.js.org/concepts#loaders
> body {
|   background-color: green;
| }
 @ ./src/index.js 4:0-20

上面报错的原因是默认情况下，webpack只认识js文件，如果是别的文件，需要使用loader转换器来转换

解析css文件需要两个loader，css-loader和style-loader，css-loader的作用就是解析css的语法，解析完成后传给style-loader， style-loader就会生成style标签，然后把这些css代码插入到html文件中，这样在html就会有样式了

接下来，我们去安装css文件相关的转换器

npm i style-loader@1.1.3  css-loader@3.4.2

修改webpack.config.js配置文件

// 每个文件都称为模块，module就是用来配置解析各种文件模块的，module里面主要涉及到几个问题，
  //  1 哪些模块文件需要被转换？ 2 用什么loader去转？ 3 用什么规则去转？
  module: {
    rules: [
      {
        test: /\.css$/,
        use: [
          'style-loader',
          'css-loader'
        ]
      }
    ]
  }

loader里面的配置主要包括下面几项：

test: 是一个正则表达式，用来匹配对应的文件
use: loader的名称
include/exclude: 手动指定必须处理的文件夹或屏蔽不需要处理的文件夹
query: 为loader提供额外的配置

注意： loader的写法有三种： 1、字符串 2 数组 3、对象

注意： css-loader和style-loader是需要注意使用顺序的，如果配置成字符串的形式，css-loader要放在下面，如果配置成数组的形式，css-loader要放在前面，顺序不正确会报错

# 7.自动给css添加前缀

在写css的时候，某些属性我们可能会增加浏览器前缀，这个功能我们可以使用postcss-loader来帮我们实现,其中postcss-loader需要一个autoprefix插件来实现自动添加前缀

安装相关loader

npm i postcss-loader@3.0.0 autoprefixer@9.7.5

在项目文件夹下，新建一个postcss.config.js来给postcss-loader增加配置

module.exports = {
  plugins: [require("autoprefixer")]
}

接下来，需要在项目下，新建一个.browserslistrc文件，配置相关浏览器支持情况

// 项目要覆盖95%以上浏览器, 注意，这里的注释不要跟着写到你自己到配置文件里面，会报错
cover 95%

修改webpack.config.js

{
  test: /\.css$/,
  use: [
    "style-loader",
    {
      loader: "css-loader",
      options: {
        // css文件解析后，需要用后面的1个loader来处理，如果后面有多个loader，需要更改一下importLoaders的数量
        importLoaders: 1
      }
    },
    // "css-loader",
    "postcss-loader"
  ]
},

# 8.单独抽离css文件

前面我们用style-loader把打包的css代码都插入到html的style标签里，如果我们想把css文件单独打包到某个文件中，再通过外联的样式来使用，这种需求，我们需要通过mini-css-extract-plugin插件来实现

安装

npm i mini-css-extract-plugin@0.9.0

修改配置文件

// 引入nodejs的path模块来处理路径
const path = require('path')
const htmlWebpackPlugin = require('html-webpack-plugin')
const { CleanWebpackPlugin } = require('clean-webpack-plugin')
const MiniCssExtractPlugin = require('mini-css-extract-plugin')
module.exports = {
  // 模式配置
  mode: 'production',
  // 入口文件  通常使用绝对路径
  entry: path.resolve(__dirname, './src/index.js'),
  // 出口配置
  output: {
    filename: 'app.js',
    path: path.resolve(__dirname, 'dist')
  },
  devServer: {
    contentBase: path.resolve(__dirname, 'dist'),
    open: true
  },
  plugins: [
    new MiniCssExtractPlugin({
      // 配置一下抽离出来的css文件名
      filename: 'css/main.css'
    }),
    new htmlWebpackPlugin({
      // 设置标题
      title: '螺钉课堂',
      // 设置模版路径
      template: path.resolve(__dirname, 'public/index.html'),
      // 设置打包的文件名
      filename: 'index.html',
      // 最小输出配置
      minify: {
        // 折叠空格
        collapseWhitespace: true
      }
    }),
    // 每次打包前 先清空上次打包的目录
    new CleanWebpackPlugin()
  ],
  module: {
    rules: [
      {
        test: /\.css$/,
        loader: [
          MiniCssExtractPlugin.loader,
          {
            loader: 'css-loader',
            options: {
              // css文件解析后，需要用后面的1个loader来处理，如果后面有多个loader，需要更改一下importLoaders的数量
              importLoaders: 1
            }
          },
          'postcss-loader'
        ]
      }
    ]
  }
}

# 9.压缩css

默认情况下，webpack只能压缩js而不能压缩css，如果想压缩css，需要单独使用插件来完成

安装插件

npm i optimize-css-assets-webpack-plugin@5.0.3

修改配置文件

// 引入nodejs的path模块来处理路径
const path = require('path')
const htmlWebpackPlugin = require('html-webpack-plugin')
const { CleanWebpackPlugin } = require('clean-webpack-plugin')
const MiniCssExtractPlugin = require('mini-css-extract-plugin')
const OptimizeCSSAssetsPlugin = require('optimize-css-assets-webpack-plugin')
module.exports = {
  // 模式配置
  mode: 'production',
  // 入口文件  通常使用绝对路径
  entry: path.resolve(__dirname, './src/index.js'),
  // 出口配置
  output: {
    filename: 'app.js',
    path: path.resolve(__dirname, 'dist')
  },
  devServer: {
    contentBase: path.resolve(__dirname, 'dist'),
    open: true
  },
  plugins: [
    new MiniCssExtractPlugin({
      // 配置一下抽离出来的css文件名
      filename: 'css/main.css'
    }),
    new htmlWebpackPlugin({
      // 设置标题
      title: '螺钉课堂',
      // 设置模版路径
      template: path.resolve(__dirname, 'public/index.html'),
      // 设置打包的文件名
      filename: 'index.html',
      // 最小输出配置
      minify: {
        // 折叠空格
        collapseWhitespace: true
      }
    }),
    // 每次打包前 先清空上次打包的目录
    new CleanWebpackPlugin()
  ],
  module: {
    rules: [
      {
        test: /\.css$/,
        loader: [
          MiniCssExtractPlugin.loader,
          {
            loader: 'css-loader',
            options: {
              // css文件解析后，需要用后面的1个loader来处理，如果后面有多个loader，需要更改一下importLoaders的数量
              importLoaders: 1
            }
          },
          'postcss-loader'
        ]
      }
    ]
  },
  optimization: {
    // 配置压缩方案
    minimizer: [
      // 配置css压缩方案，注意，css手动选择压缩方式后，webpack就不会帮助压缩js文件了，也需要手动配置
      new OptimizeCSSAssetsPlugin({})
    ]
  }
}

安装了这个插件后，js压缩失效，因此需要单独安装js压缩插件

npm i terser-webpack-plugin@2.3.5

# 10.预处理器文件处理

# 解析 sass文件

sass这种css预处理器是以.scss结尾，需要用node-sass和sass-loader来处理

安装loader

npm i node-sass@4.13.1 sass-loader@8.0.2

增加相关配置：

module: {
  rules: [
    {
      test: /\.css$/,
      use: ["style-loader", "css-loader"]
    },
    {
      test: /\.scss$/,
      use: ["style-loader", "css-loader", "sass-loader"]
    }
  ]
}

注意： 如果出现css文件中引入sass文件的情况，只用css-loader是不能解析的，必须加入sass-loader，并且需要在css-loader中声明，使用sass-loader才可以

举个例子： index.css 引入 a.css , a.css引入a.scss，这个时候需要增加配置才行

{
  test: /\.css$/,
  use: [
    "style-loader",
    // 需要把原来的字符串换成对象的形式，然后给css-loader配置参数
    {
      loader: "css-loader",
      options: {
        // 如果css文件引入其他文件(@import), 使用后面的1个loader来处理
        importLoaders: 2
      }
    },
    // "css-loader",
    "postcss-loader",
    "sass-loader"
  ]
},

# 11.解析图片

解析图片需要用到file-loader，它的作用是将图片拷贝到打包后到输出目录，并且返回一个可以引用的地址

安装file-loader

npm i file-loader

增加图片支持配置

{
  test: /\.(jpe?g|png|gif|bmp)$/,
  use: "file-loader"
}

当图片比较小的时候，我希望把这个图片转成base64格式的字符串，这样的好处是少发http请求，完成这个转base64字符串需求需要用到url-loader， url-loader是在file-loader上做的一层封装， 我们需要将这个loader先安装上

npm i url-loader

修改相关配置

{
  test: /\.(jpe?g|png|gif|bmp|svg)$/,
  use: {
    loader: "url-loader",
    options: {
      // 8k以下的图片转成base64
      limit: 8192,
      name: "img/[name].[ext]"
    }
  }
}

如果要支持字体图标，可以再新建一条匹配规则，使用file-loader来处理即可

增加相关配置

{
  test: /\.(eof|ttf|woff|woff2)$/,
  use: "file-loader"
}

# 12.解析js文件

ES的一些比较新的语法浏览器是不支持的，因此，我们需要将这些新的语法转成比较通用的语法，babel就是一个很好的转换工具，我们先安装需要的工具

npm i @babel/core@7.9.0 babel-loader@8.1.0 @babel/preset-env@7.9.0

@babel/core是babel的核心，它转换语法的时候会用到 @babel/preset-env这个插件包， @babel/preset-env这些插件包里包含了各种新语法的转换功能，babel这个工具是可以独立运行的，如果想要和webpack结合，还需要安装一个babel-loader， babel-loader的作用就是把ES的一些新语法送到@babel/core，@babel/core再去调用@babel/preset-env插件来完成转换

增加相关配置

{
  test: /\.js$/,
  use: "babel-loader"
}

babel-loader 会去调一个babel的配置文件，我们需要在项目根目录下创建.babelrc的配置文件

{
  "presets": ["@babel/preset-env"]
}

如果使用了一些@babel/preset-env不能转换的语法，你还可以单独安装对应的转换插件来解决，例如：

class Person {
  name = "小红"
}

这种语法需要用 @babel/plugin-proposal-class-properties 这个插件来转换 安装插件

npm i @babel/plugin-proposal-class-properties@7.8.3
