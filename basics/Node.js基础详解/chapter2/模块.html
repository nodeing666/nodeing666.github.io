# 2.模块

# 2.1.什么是模块以及注意事项？

在nodejs中，一个文件就是一个模块,每个模块都有自己独立的作用域

注意1：在node环境中，全局对象并不是window，而是global

// 在node环境中输出window 会报错
console.log(window)

注意2: 在node环境中，通过var定义的变量都是模块内局部的变量

// 模块内局部变量
var num = 100
// 定义全局变量 
global.num = 200

console.log(num)  // 100
console.log(global.num) //200

# 2.2.Node中模块的加载细节

在node中加载模块是通过require这个方法完成的

// 加载b模块
var b = require('./b.js')

注意1：加载模块使用相对路径的时候，一定不要省略 "./", 也就是说 "./b.js" 和 “b.js” 加载的规则是不同的

require('./b.js')  //加载的是当前目录下的b.js模块

require('b.js')  // 加载的是node的核心模块(内置模块)或者第三方模块(node_modules文件夹下的模块)

注意2；模块文件查找机制,如果引入模块的时候省略扩展名，会有相应的文件查找机制

require('./b')

上面引入一个b模块，查找文件的顺序如下:

./b    1 优先查找没有后缀的b文件
./b.js 2 如果第一步没有找到，就查找b.js文件
./b.json 3 如果上一步没有找到，就查找b.json文件
./b.node  4 如果上一步没找到就查找b.node文件
5 如果上面几步都没找到，那么会报错

# 2.3.导出模块

思考：在一个模块中定义一个变量，如何在另一个模块中使用？

解决方案一： 在模块中定义全局变量

b.js模块

global.num = 200

a.js模块中使用

require('./b')
console.log(global.num)

由于我们需要尽可能的减少全局变量污染，因此，这种解决方案并不推荐

解决方案二： 使用module对象导出内容

module对象为模块对象，保存了当前模块有关的信息

我们可以打印出来看看：

console.log(module)

内容如下：

在module对象中，有一个非常重要的对象（exports对象），这个对象中的内容就是对外暴露的

b.js模块

var num = 100
// 把num暴露出去供别人访问
module.exports.num = num

a.js模块

// require方法的返回值，就是b模块中module.exports对象
var b = require('./b')
console.log(b)

# 2.4.CommonJS 模块规范

先说核心关键字 “规范”，什么是规范呢？通俗的说就是明文规定或约定俗成的标准，CommonJS模块规范就是一种JS模块的约定或者标准，它主要规定的以下几个问题:

1. 模块如何定义？ 每个文件就是一个模块

2. 模块如何导出？ module.exports

3. 模块如何使用？ require

# 为什么会有模块化

1.解决全局变量污染

2.方便维护

3.方便协作

通过声明一个对象的形式来实现模块化

var obj = {
  name: "坤坤"
}
var obj = {
  name: "菜菜"
}

通过闭包的形式来实现模块化

(function(){
  var a = 10
  function fn(){
    console.log(a)
  }
  return fn
})()

# 如何让一个字符串执行？

1.使用eval方式执行

md1.js

var a = 10
console.log(b)
module.exports = a

test.js

const fs = require('fs')
const path = require('path')

fs.readFile(path.resolve(__dirname, './md1.js'), 'utf-8', function(err, data) {
  var b = 10
  eval(data)
})

从上面例子中，我们可以看出，在md1模块中可以使用，外部模块的变量，eval函数并不会产生一个新的作用域，它的作用域就是当前的作用域，因此，可能出现模块之间变量的冲突

2.使用New Function的方式执行

const fs = require('fs')
const path = require('path')

fs.readFile(path.resolve(__dirname, './md1.js'), 'utf-8', function(err, data) {
  var b = 10
  // eval(data)
  var c = new Function(data)
  console.log(c())
})

这种方式会在执行字符串外面包一层匿名函数，这样会形成一个私有的作用域，模块之间不会影响, 通常使用在模版引擎中

3.使用Nodejs中的vm模块方式执行

VM 模块是 NodeJS 里面的核心模块，支撑了 require 方法和 NodeJS 的运行机制，可以使用vm模块来创建独立运行的沙箱体制

vm.runInThisContext(code, filename);

此方法用于创建一个独立的沙箱运行空间，code 内的代码可以访问外部的 global 对象，但是不能访问其他变量 而且 code 内部 global 与外部共享


var vm = require('vm')

var a = 6
// 全局变量a
global.a = 999
var b = 10
vm.runInThisContext("console.log('ok', a)") // 显示global下的999
// vm.runInThisContext('console.log(global)') // 显示global
vm.runInThisContext('console.log(b)') // 报错 ，不能访问全局变量外的其他变量

console.log(a)
